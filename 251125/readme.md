🔷 파이썬 객체(Object)

객체는 “현실 세계의 사물”을 프로그램으로 표현한 것.

**속성(값)**과 **행동(메서드)**을 함께 가진다.

파이썬에서는 모든 데이터가 객체이므로,

숫자도 메서드를 가지고 있고 문자열도 객체처럼 동작한다.

예:

```python
x = 5
print(x.bit_length())   # 정수 객체의 메서드 호출
```

🔷 파이썬 클래스(Class)

클래는 객체를 만들기 위한 설계도(템플릿).

객체는 클래스를 기반으로 생성된 인스턴스.

클래스 안에는 **속성(변수)**과 **메서드(함수)**를 정의한다.

클래스 안에서 미리 속성과 행동을 정의해두고,

이를 활용해 동일한 구조의 객체를 여러 개 만들 수 있다.

예:

```python
class Dog:
    def __init__(self, name, age):  # 속성 정의
        self.name = name
        self.age = age
    
    def bark(self):                 # 행동 정의
        return f"{self.name} is barking!"

d1 = Dog("Coco", 3)    # Dog 클래스의 객체(인스턴스)
```

🔷 절차지향 프로그래밍 (Procedural Programming)

프로그램을 **순서(절차)**대로 처리하는 방식.

데이터와 함수가 분리되어 있다.

작은 규모의 프로그램에서 이해·작성하기 쉬움.

예: "입력 → 처리 → 출력" 형태의 순차적 코드

특징

흐름 중심

구조 단순

유지보수는 규모가 커질수록 어려움

관련 언어: C, BASIC, Fortran, Pascal

---

🔷 객체지향 프로그래밍 (Object-Oriented Programming)

프로그램을 객체들의 상호작용으로 구성하는 방식.

데이터와 기능을 하나로 묶은 객체 중심.

재사용성, 확장성, 유지보수성이 좋음.

주요 개념

캡슐화: 데이터 + 메서드를 묶음

상속: 기존 클래스를 기반으로 새로운 클래스 생성

다형성: 동일한 메서드 이름으로 다른 동작 가능

관련 언어: Python, Java, C++, C#, Ruby, Swift, Kotlin

---

🔷 절차지향 vs 객체지향 요약 비교
| 구분 | 절차지향 | 객체지향 |
|--------|--------|------|
| 중심 | 함수, 절차 | 객체 |
| 구조 | 단순 | 복잡하지만 체계적 |
| 재사용성 | 낮음 | 높음 |
| 유지보수 | 어려움 | 쉬움 |
| 적합한 규모 | 작은 프로젝트 | 중·대규모 프로젝트 |
