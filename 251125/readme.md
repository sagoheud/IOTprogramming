🔷 파이썬 객체(Object)

파이썬에서 모든 것은 객체이다. (숫자, 문자열, 함수, 리스트 등)

객체는 **값 + 타입 + 메서드(행동)**을 가진다.

예:

a = 10      # int 객체
b = "hi"    # str 객체

---

🔷 파이썬 클래스(Class)

클래는 객체를 만들기 위한 설계도(템플릿).

객체는 클래스를 기반으로 생성된 인스턴스.

클래스 안에는 **속성(변수)**과 **메서드(함수)**를 정의한다.

예시
class Person:
    def __init__(self, name):
        self.name = name
    
    def greet(self):
        print(f"Hello, I'm {self.name}")

p = Person("Tom")  # 인스턴스 생성
p.greet()

🔷 절차지향 프로그래밍 (Procedural Programming)

프로그램을 **순서(절차)**대로 처리하는 방식.

데이터와 함수가 분리되어 있다.

작은 규모의 프로그램에서 이해·작성하기 쉬움.

예: "입력 → 처리 → 출력" 형태의 순차적 코드

특징

흐름 중심

구조 단순

유지보수는 규모가 커질수록 어려움

---

🔷 객체지향 프로그래밍 (Object-Oriented Programming)

프로그램을 객체들의 상호작용으로 구성하는 방식.

데이터와 기능을 하나로 묶은 객체 중심.

재사용성, 확장성, 유지보수성이 좋음.

주요 개념

캡슐화: 데이터 + 메서드를 묶음

상속: 기존 클래스를 기반으로 새로운 클래스 생성

다형성: 동일한 메서드 이름으로 다른 동작 가능

---

🔷 절차지향 vs 객체지향 요약 비교
| 구분 | 절차지향 | 객체지향 |
|--------|--------|------|
| 중심 | 함수, 절차 | 객체 |
| 구조 | 단순 | 복잡하지만 체계적 |
| 재사용성 | 낮음 | 높음 |
| 유지보수 | 어려움 | 쉬움 |
| 적합한 규모 | 작은 프로젝트 | 중·대규모 프로젝트 |
