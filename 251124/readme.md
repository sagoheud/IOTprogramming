## 🚀 필수 준비 및 핵심 기능

VS Code에서 파이썬을 사용하기 위해 가장 먼저 필요한 것은 **Microsoft 공식 Python 확장(Extension)**을 설치하는 것입니다. 이 확장을 설치하면 다음과 같은 핵심 기능들을 사용할 수 있습니다.

| 기능 | 설명 |
| :--- | :--- |
| **인터프리터 선택/전환** | 여러 가상 환경(venv, conda 등)이나 설치된 파이썬 버전($3.x$ 등)을 쉽게 감지하고 선택할 수 있습니다. |
| **코드 편집 보조** | **IntelliSense(Pylance)**를 통해 자동 완성, 함수 정의 보기, 코드 조각(snippets) 제공 등의 기능을 지원하여 코딩 속도와 정확도를 높입니다. |
| **디버깅** | 코드에 중단점(breakpoint)을 설정하고, 변수 값을 실시간으로 확인하며 코드를 단계별로 실행하여 오류를 쉽게 찾고 수정할 수 있습니다. |
| **코드 실행** | 작성한 `.py` 파일을 터미널을 이용하거나, 편집기 우측 상단의 '▶ (실행 버튼)' 클릭 한 번으로 빠르게 실행할 수 있습니다. |
| **코드 분석 및 포맷팅** | **린팅(Linting)** 기능을 통해 잠재적인 버그나 스타일 오류를 미리 알려주며, **포맷팅(Formatting)** 도구(예: Black, autopep8)를 이용해 코드를 PEP 8 표준에 맞게 자동으로 정리할 수 있습니다. |
| **Jupyter Notebook 지원** | Jupyter 확장과 함께 사용하면 VS Code 내에서 `.ipynb` 파일을 생성, 편집 및 실행할 수 있어 데이터 과학 작업에 특히 유용합니다. |

## 🛠️ 기본적인 사용 흐름

1.  **Python 설치:** 먼저 시스템에 파이썬 인터프리터가 설치되어 있어야 합니다.
2.  **VS Code 확장 설치:** VS Code의 확장 뷰(왼쪽 사이드바의 네모 아이콘)에서 "**Python**"을 검색하여 Microsoft에서 제공하는 공식 확장을 **설치**합니다.
3.  **프로젝트 폴더 열기:** `파일(File)` > `폴더 열기(Open Folder)`를 통해 작업할 프로젝트 폴더를 엽니다.
4.  **인터프리터 선택:** VS Code 왼쪽 하단에 표시되는 파이썬 인터프리터를 클릭하여 프로젝트에 맞는 **가상 환경**이나 파이썬 버전을 선택합니다.
5.  **코딩 및 실행:** `.py` 파일을 작성하고, 오른쪽 상단의 실행 버튼이나 단축키(예: `Ctrl + F5` 또는 `F5`)를 눌러 코드를 실행합니다. 결과는 VS Code 통합 터미널에 표시됩니다.

VS Code는 이러한 강력한 기능들을 제공함으로써 파이썬 개발을 효율적이고 생산적으로 만들어 줍니다.

## ⚙️ VS Code 파이썬 환경 설정 및 사용 (간단 설명)

| 단계 | 목적 | 간단 설명 |
| :--- | :--- | :--- |
| **1. 확장 설치** | VS Code에서 파이썬 기능 활성화 | 확장 뷰(Ctrl+Shift+X)에서 **"Python"** (Microsoft)을 검색해 설치합니다. |
| **2. 파이썬 설치** | 코드를 실행할 인터프리터 준비 | PC에 Python.org에서 최신 버전을 설치합니다. (VS Code가 아님) |
| **3. 가상 환경 생성 (권장)** | 프로젝트별 환경 격리 | 터미널에서 `python -m venv .venv` 명령으로 **독립된 환경**을 만듭니다. |
| **4. 인터프리터 선택** | VS Code에 사용할 파이썬 지정 | VS Code 오른쪽 아래의 파이썬 버전을 클릭하여, 방금 만든 `.venv`의 파이썬을 선택합니다. |
| **5. 코딩 보조** | 코드 작성 속도 및 정확도 향상 | **Pylance**가 자동 완성, 오류 검사, 정의 이동(Ctrl + 클릭) 등을 제공합니다. |
| **6. 실행 및 디버깅** | 코드 동작 확인 및 오류 수정 | `.py` 파일을 열고, 오른쪽 상단 '▶ (실행)' 버튼으로 실행하거나, `F5`를 눌러 중단점을 설정한 상태에서 디버깅을 시작합니다. |

**핵심:** **Python 확장** 설치 후, 프로젝트에 맞는 **인터프리터(가급적 가상 환경)**를 선택하는 것이 가장 중요합니다! 👍
